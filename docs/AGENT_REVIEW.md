**Findings**
1. `Blocker:` a real Google Maps API key was committed into [android/gradle.properties](C:/Users/T/Desktop/StallPass/android/gradle.properties:68). That violates the repo’s secret-handling contract and bypasses the existing env-based production guard in [app.config.ts](C:/Users/T/Desktop/StallPass/app.config.ts:63) and [android/app/build.gradle](C:/Users/T/Desktop/StallPass/android/app/build.gradle:124). This key needs to be rotated and removed from VCS, not shipped from Gradle properties.

2. `High:` guest and pre-auth crashes still cannot be reported. [src/api/bug-reports.ts](C:/Users/T/Desktop/StallPass/src/api/bug-reports.ts:45) treats “no session” as a terminal failure, [src/components/ErrorBoundary.tsx](C:/Users/T/Desktop/StallPass/src/components/ErrorBoundary.tsx:99) only queues when it can recover a `userId`, and [src/hooks/useOfflineSync.ts](C:/Users/T/Desktop/StallPass/src/hooks/useOfflineSync.ts:349) never initializes or replays the queue without `user?.id`. The edge function was written to allow `user_id = null`, but the client path cannot reach that branch.

3. `High:` the promised “5 reports per device per hour” protection is not actually implemented. The payload built in [src/api/bug-reports.ts](C:/Users/T/Desktop/StallPass/src/api/bug-reports.ts:14) and typed in [src/types/index.ts](C:/Users/T/Desktop/StallPass/src/types/index.ts:115) has no `device_id`, so [supabase/functions/submit-bug-report/index.ts](C:/Users/T/Desktop/StallPass/supabase/functions/submit-bug-report/index.ts:103) falls back to `userId` or `'unknown'`. One physical device can bypass the quota by switching accounts, and any future guest flow would collapse into a shared global bucket.

4. `High:` the dashboard script will throw on first load. [web/assets/business.js](C:/Users/T/Desktop/StallPass/web/assets/business.js:205) reads `wasAllComplete`, but [web/assets/business.js](C:/Users/T/Desktop/StallPass/web/assets/business.js:406) calls `render()` before [web/assets/business.js](C:/Users/T/Desktop/StallPass/web/assets/business.js:539) initializes `let wasAllComplete`. That is a TDZ `ReferenceError` during module evaluation.

5. `Medium:` the Fresco pin in [android/app/build.gradle](C:/Users/T/Desktop/StallPass/android/app/build.gradle:224) regresses animated image modules to `3.1.3`, while the current React Native toolchain in [node_modules/react-native/gradle/libs.versions.toml](C:/Users/T/Desktop/StallPass/node_modules/react-native/gradle/libs.versions.toml:24) is on `3.6.0`. If this was meant as a build fix, it should match the repo’s resolved version or stay template-driven; otherwise you’re introducing manual native dependency skew.
